# 0.2.0 Plan

## Goal

Keep `PPath(...)` as a local-first API that enters remote mode only when the caller explicitly provides remote configuration.

For `0.2.0`, `PPath(...)` should support:

- local paths with no configuration
- named profile-based remote configuration
- one-shot inline remote configuration
- explicit `s3://...` and `gs://...` URIs only when remote configuration is already present

This plan intentionally excludes:

- Azure support
- new exception hierarchy
- `gcs://` alias support
- generic `http(s)` handling in `PPath(...)`
- no-profile remote convenience such as `PPath("s3://...")`
- public URL canonicalization inside `PPath(...)`

URI-first remote usage remains the responsibility of existing `CloudPath`, `HttpPath`, and `HttpsPath` entry points.

## API Direction

### Supported Call Patterns

Local mode:

```python
from ppathlib import PPath

path = PPath("data/local-report.parquet")
```

Profile mode:

```python
from ppathlib import PPath

path = PPath("daily/report.parquet", profile="MY_BUCKET")
```

Inline one-shot configuration mode:

```python
from ppathlib import PPath

path = PPath(
    "daily/report.parquet",
    config={
        "STORAGE_TYPE": "s3",
        "ROOT": "s3://demo-bucket/data",
        "ACCESS_KEY_ID": "xxx",
        "SECRET_ACCESS_KEY": "yyy",
        "ENDPOINT_URL": "https://example.r2.cloudflarestorage.com",
        "REGION": "auto",
    },
)
```

Configured explicit remote URI:

```python
from ppathlib import PPath

path = PPath(
    "s3://demo-bucket/data/file.parquet",
    config={
        "STORAGE_TYPE": "s3",
        "ACCESS_KEY_ID": "xxx",
        "SECRET_ACCESS_KEY": "yyy",
        "ENDPOINT_URL": "https://example.r2.cloudflarestorage.com",
    },
)
```

URI-first remote usage stays separate:

```python
from ppathlib import CloudPath

path = CloudPath("https://example.com/files/report.csv")
```

### Public API Additions

Add runtime profile registration for reusable named configuration:

```python
register_profile(profile: str, *, config: Mapping[str, str | os.PathLike[str] | None]) -> None
unregister_profile(profile: str) -> None
clear_profile_registry() -> None
```

### Existing Public API Constraints

Keep `get_client(...)` as a profile-based public API:

```python
get_client(profile: str) -> Any
```

Do not add `get_client(config=...)` in `0.2.0`.

### Mode Rules

- `profile` and `config` are mutually exclusive.
- If both are omitted, `PPath(...)` returns a local `Path`.
- `profile` is for reusable named configuration.
- `config` is for one-shot flat configuration.
- `config` uses canonical ppathlib-style keys in documentation.
- Compatibility aliases may be accepted internally, but they are not the primary documented API.
- Explicit remote URIs are valid only in remote mode.
- Relative remote paths require `ROOT`.
- `PPath(...)` does not become a scheme-driven URI dispatcher in `0.2.0`.

## Layered Design

### Layer 1: Source Resolution

Layer 1 lives in `ppathlib/ppath.py` and resolves configuration into a storage-aware source.

Responsibilities:

- validate `profile` vs `config`
- normalize inline `config`
- merge runtime registered profile values over `os.environ`
- canonicalize storage type
- resolve backend client settings
- resolve optional `ROOT`

Expected Layer 1 output:

- resolved storage type
- resolved root
- resolved path class
- resolved client kwargs

### Layer 1.5: Path Resolution

Path resolution stays separate from source resolution.

Responsibilities:

- accept explicit remote URI only when remote mode is active
- ensure URI scheme matches resolved storage type
- resolve relative paths against `ROOT`
- reject local-looking paths in remote mode when `ROOT` is missing

Expected output:

- final remote path string

### Layer 2: Backend-Specific Handling

Layer 2 remains backend-specific and should preserve current backend behavior:

- S3 path/client construction
- GCS path/client construction
- existing HTTP path/client behavior outside `PPath(...)`

Layer 2 should not know whether configuration came from:

- `os.environ`
- runtime registered profile values
- per-call inline `config`

## Work Items

## 1. Refactor Source Resolution

Split source resolution from path resolution in `ppathlib/ppath.py`.

Implementation targets:

- add a source resolver for `profile` or `config`
- add a config normalization helper
- keep path resolution separate from client/source resolution
- keep `PPath(...)` and `get_client(...)` aligned on source resolution only

Key decisions:

- `profile` mode reads runtime registry first, then `os.environ`
- inline `config` mode bypasses registry and `os.environ`
- storage type can be inferred from `ROOT` or explicit remote URI in `PPath(...)`
- if storage type cannot be inferred, `STORAGE_TYPE` is required
- `get_client(...)` does not participate in path inference

Use cases:

```python
PPath("daily/report.parquet", profile="MY_BUCKET")
PPath("daily/report.parquet", config={"STORAGE_TYPE": "s3", "ROOT": "s3://bucket/root"})
```

## 2. Add Runtime Profile Registration

Add a process-local registry for reusable profile configuration.

Behavior:

- registration stores flat canonical config by normalized profile name
- registered values override `os.environ`
- unregister removes only the runtime registration
- clearing the registry removes all runtime registrations
- registry changes must not force a global client cache reset

Use case:

```python
from ppathlib import register_profile, PPath

register_profile(
    "R2_DEV",
    config={
        "STORAGE_TYPE": "s3",
        "ROOT": "s3://demo-bucket/data",
        "ENDPOINT_URL": "https://example.r2.cloudflarestorage.com",
        "ACCESS_KEY_ID": "xxx",
        "SECRET_ACCESS_KEY": "yyy",
        "REGION": "auto",
    },
)

path = PPath("daily/report.parquet", profile="R2_DEV")
```

## 3. Add Inline `config` Mode

Support flat per-call configuration through `config`.

Rules:

- no `profile` required
- no nested config objects
- documentation shows canonical keys only
- compatibility aliases may be normalized internally before backend resolution

Use cases:

```python
PPath(
    "daily/file.parquet",
    config={
        "STORAGE_TYPE": "s3",
        "ROOT": "s3://demo-bucket/data",
        "ENDPOINT_URL": "https://example.r2.cloudflarestorage.com",
        "ACCESS_KEY_ID": "xxx",
        "SECRET_ACCESS_KEY": "yyy",
    },
)
```

```python
PPath(
    "gs://demo-bucket/data/file.parquet",
    config={
        "STORAGE_TYPE": "gs",
        "PROJECT": "demo-project",
        "CREDENTIALS_JSON": "/tmp/service-account.json",
    },
)
```

## 4. Preserve Explicit Remote URI Support Only in Remote Mode

Allow explicit `s3://...` and `gs://...` URIs only when `profile` or `config` is provided.

Behavior:

- explicit remote URI is valid in remote mode
- URI scheme must match resolved storage type
- `PPath("s3://bucket/key")` without `profile` or `config` remains invalid
- generic `http(s)` input remains outside `PPath(...)`

Use cases:

```python
PPath(
    "s3://demo-bucket/data/file.parquet",
    profile="R2_DEV",
)
```

```python
PPath(
    "gs://demo-bucket/data/file.parquet",
    config={
        "STORAGE_TYPE": "gs",
        "PROJECT": "demo-project",
        "CREDENTIALS_JSON": "/tmp/service-account.json",
    },
)
```

## 5. Define Cache and `get_client(...)` Semantics

Keep client caching based on canonical effective client settings.

Rules:

- client cache keys include storage type and effective client kwargs only
- `ROOT` is not part of the cache key
- profile name is not part of the cache key unless it affects actual client kwargs
- alias differences must normalize to the same effective cache key
- registry updates must not clear unrelated cached clients
- `get_client(...)` stays public profile-only
- internal source resolution may be shared with `PPath(...)`

Use cases:

```python
client_a = get_client("PROFILE_A")
client_b = get_client("PROFILE_B")

assert client_a is client_b
```

The above remains valid when `PROFILE_A` and `PROFILE_B` differ only by `ROOT`.

## 6. Add Tests

Primary target: `ppath_tests/test_ppath.py`

Coverage to add:

- reject `profile` and `config` together
- resolve inline `config` for relative path plus `ROOT`
- resolve inline `config` for explicit S3 and GCS URIs
- runtime registered profile overrides `os.environ`
- unregister falls back to `os.environ`
- same effective client settings share one client even when `ROOT` differs
- root-only runtime registry changes do not rebuild the client
- compatibility aliases normalize to the same effective client settings
- `PPath("s3://bucket/key")` without configuration remains invalid
- `get_client(...)` remains profile-only

Representative test shape:

```python
def test_ppath_rejects_profile_and_config_together():
    with pytest.raises(ValueError):
        PPath(
            "daily/report.parquet",
            profile="R2_DEV",
            config={"STORAGE_TYPE": "s3"},
        )
```

```python
def test_root_only_registry_change_does_not_rebuild_client(monkeypatch, fake_s3_builder):
    register_profile(
        "R2_DEV",
        config={
            "STORAGE_TYPE": "s3",
            "ROOT": "s3://bucket-a/root",
            "ENDPOINT_URL": "https://example.r2.cloudflarestorage.com",
            "ACCESS_KEY_ID": "xxx",
            "SECRET_ACCESS_KEY": "yyy",
        },
    )

    client1 = get_client("R2_DEV")

    register_profile(
        "R2_DEV",
        config={
            "STORAGE_TYPE": "s3",
            "ROOT": "s3://bucket-b/root",
            "ENDPOINT_URL": "https://example.r2.cloudflarestorage.com",
            "ACCESS_KEY_ID": "xxx",
            "SECRET_ACCESS_KEY": "yyy",
        },
    )

    client2 = get_client("R2_DEV")

    assert client1 is client2
```

## Implementation Order

1. Add config normalization and runtime profile registry helpers.
2. Split source resolution from path resolution.
3. Add inline `config` mode.
4. Restrict explicit remote URI handling to configured remote mode.
5. Lock cache and `get_client(...)` semantics.
6. Add tests.
7. Update docs and public exports.

## Non-Goals for 0.2.0

- Azure parity
- new exception subclasses
- support for `gcs://`
- generic `http(s)` support in `PPath(...)`
- no-profile `s3://...` or `gs://...` convenience in `PPath(...)`
- public URL canonicalization in `PPath(...)`
- `get_client(config=...)`

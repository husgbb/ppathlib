# PHILOSOPHY

- `ppathlib` must provide a `pathlib.Path`-like experience for remote paths.
- `ppathlib` must keep its public path interface backend-neutral.
- `ppathlib` must stay lightweight and fast.
- `ppathlib` must support project-scoped configuration.
- `ppathlib` must store configuration in files.
- `ppathlib` must keep lexical path operations network-free.
- `ppathlib` must fail explicitly when behavior is not implemented.
- `ppathlib` must keep tests intuitive and easy for humans to read.
- `ppathlib` must treat tests as contract verification.
